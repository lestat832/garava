Garava

Selective Garmin-to-Strava Activity Sync

Use the right data source for each activity on Strava. If you record with a cycling computer or gym app alongside a Garmin watch, Garava makes sure the better version is the one that lands on Strava.

View on GitHub Get Started

Sound familiar?

Garmin watch + Karoo. You wear a Garmin for training load and ride with a Karoo. Both record the ride. The Karoo has your power meter and richer cycling data -- that's what you want on Strava. But Garmin pushes its version too and you get a duplicate (or Strava picks the wrong one).

Garmin watch + Hevy. You wear a Garmin during gym sessions for training load but log sets and reps in Hevy. Hevy syncs detailed workout data to Strava. Garmin also syncs a generic "Strength Training" entry with nothing but heart rate. Two entries, one is useless.

The fix is the same. Garava replaces Garmin's native Strava sync and blocks the activity types you have a better source for. Everything else still syncs automatically.

How it works

Runs in the background on your Mac, Linux box or a server. Your machine needs to be awake for sync to happen — if it's asleep, it picks up when it wakes.

1

Fetch

Pulls your latest activities from Garmin Connect. Syncs on the quarter hour — :00, :15, :30, :45.

2

Filter

Checks each activity against your blocklist. Block activity types you have a better source for -- cycling (Karoo handles it), gym (Hevy handles it), whatever applies to your setup.

3

Sync

Downloads the original FIT file and uploads it to Strava. Heart rate, power, GPS — everything comes through.

What else it does

Beyond the filtering, there's some stuff under the hood that makes it work without babysitting.

You Pick the Blocklist

One environment variable, comma-separated list. Block cycling types if you ride with a Karoo, gym types if you log with Hevy, or whatever fits your setup.

Original FIT Files

Uploads the actual FIT file from Garmin, not a summary. Heart rate, power, GPS — it all comes through.

No Duplicates

Tracks every activity in a local database. If it already synced something, it won't sync it again.

Handles Its Own Errors

If a token expires or an upload fails, it sorts itself out. Failed activities get retried next cycle.

Run However You Want

One-off sync or continuous polling. Works with launchd, systemd, cron or Docker.

Docker If You Want It

Dockerfile and docker-compose included if you want to run it on a home server or VPS.

Get started

You'll need Python 3.9+, a Strava API application and your Garmin credentials. You also need to disable Garmin's native Strava sync first -- Garava replaces it entirely, so your runs, hikes and swims still get synced.

# Clone and install
$ git clone https://github.com/lestat832/garava.git && cd garava
$ pip install -e .
 
# Configure your Strava API credentials
$ cp .env.example .env && nano .env
 
# Authenticate with Garmin + Strava
$ garava setup
 
# Sync your activities
$ garava run --once
