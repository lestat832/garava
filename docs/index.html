Garava

Selective Garmin-to-Strava Activity Sync

Pick which Garmin activities get synced to Strava and block the ones you don't want. Useful if you record with a cycling computer alongside a Garmin watch.

View on GitHub Get Started

Garmin's sync doesn't let you choose

Garmin Connect pushes everything to Strava. Gym sessions, indoor rides, sauna, breathwork — all of it. There's no filter.

And if you record rides with a cycling computer too, you get duplicates on Strava — one from the watch, one from the computer.

Garava replaces that native sync. You tell it which activity types to skip (e.g. strength training, indoor cycling) and it only uploads the rest.

example blocklist: gym + indoor rides
Running              → Strava
Strength Training    → Blocked
Outdoor Cycling     → Strava
Indoor Cycling      → Blocked
Trail Run           → Strava
Yoga                → Strava
Open Water Swim     → Strava
Virtual Ride        → Blocked

How it works

Runs in the background on your Mac, Linux box or a server. Your machine needs to be awake for sync to happen — if it's asleep, it picks up when it wakes.

1

Fetch

Pulls your latest activities from Garmin Connect. Checks for new ones every 10 minutes by default.

2

Filter

Checks each activity against your blocklist. You decide what to block — strength training, indoor cycling, whatever doesn't belong on Strava for you.

3

Sync

Downloads the original FIT file and uploads it to Strava. Heart rate, power, GPS — everything comes through.

What else it does

Beyond the filtering, there's some stuff under the hood that makes it work without babysitting.

You Pick the Blocklist

One environment variable, comma-separated list of activity types. Block as many or as few as you want.

Original FIT Files

Uploads the actual FIT file from Garmin, not a summary. Heart rate, power, GPS — it all comes through.

No Duplicates

Tracks every activity in a local database. If it already synced something, it won't sync it again.

Handles Its Own Errors

If a token expires or an upload fails, it sorts itself out. Failed activities get retried next cycle.

Run However You Want

One-off sync or continuous polling. Works with launchd, systemd, cron or Docker.

Docker If You Want It

Dockerfile and docker-compose included if you want to run it on a home server or VPS.

Get started

You'll need Python 3.9+, a Strava API application and your Garmin credentials. You also need to disable Garmin's native Strava sync first — otherwise both will push activities and you'll get duplicates.

# Clone and install
$ git clone https://github.com/lestat832/garava.git && cd garava
$ pip install -e .
 
# Configure your Strava API credentials
$ cp .env.example .env && nano .env
 
# Authenticate with Garmin + Strava
$ garava setup
 
# Sync your activities
$ garava run --once
